RULES:
1. Output exactly ONE query with ONE RETURN at the end.
2. NEVER use UNION ALL. NEVER output multiple queries.
3. Always return full nodes and relationships: RETURN a, r, b
4. CRITICAL: Always capture relationships with a variable in square brackets: [r], [r:TYPE], or [r:TYPE1|TYPE2]
5. Use variable-length relationship patterns or WHERE clauses to cover multiple types.
6. Add LIMIT 100 at the end.

CRITICAL EXAMPLES - WRONG vs RIGHT:

WRONG (relationship variable not captured):

ERROR: Variable r not defined

RIGHT (relationship always has a variable):
MATCH (p:Psychiatrist)-[r:PRESCRIBED]->(m:Medication) RETURN p, r, m
✓ Correct - r captures the relationship
